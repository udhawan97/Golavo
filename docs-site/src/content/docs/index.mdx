  <section class="gv-chapter gv-chapter--architecture gv-snap" data-reveal aria-labelledby="gv-local-data-title">
    <div class="gv-chapter-copy gv-chapter-copy--wide">
      <p class="gv-kicker">Fresh when verified. Useful when offline.</p>
      <h2 id="gv-local-data-title" class="gv-chapter-title">Local data that keeps<br/><em>its receipts and its boundaries.</em></h2>
      <div class="gv-body">
        Refresh approved CC0 sources manually or with launch/while-open consent. Followed
        matches get targeted checks and deduplicated change history. OpenLigaDB stays an
        optional, display-only ODbL overlay; corrections and researched facts begin as
        untrusted local candidates. None of these paths can rewrite a sealed forecast.
      </div>
      <div class="gv-mini-steps" aria-label="Local data workflow summary">
        <span><b>Refresh</b> raw hashes, schema gates, atomic last-known-good activation</span>
        <span><b>Follow</b> kickoff, venue, score and settlement checks while the app is open</span>
        <span><b>Review</b> evidence-backed candidates; conflicts stay closed</span>
        <span><b>Isolate</b> OpenLigaDB ODbL bytes never join CC0 or model data</span>
      </div>

      <p class="gv-kicker">When a preview becomes a promise</p>
      <h2 id="gv-match-title" class="gv-chapter-title">One claim.<br/>Two receipts.<br/><em>No convenient amnesia.</em></h2>
      <div class="gv-body">
        A cockpit preview is exploratory until you seal it. The seal freezes the

  <section class="gv-chapter gv-chapter--intelligence gv-snap" data-reveal aria-labelledby="gv-ai-title">
    <div class="gv-chapter-copy">
      <p class="gv-kicker">A clear authority line</p>
      <h2 id="gv-ai-title" class="gv-chapter-title">The engine decides.<br/><em>AI may explain.</em></h2>
      <div class="gv-body">
        Deterministic code produces the probabilities and score matrix. The
        Commentator’s Notebook turns fixed, source-backed facts into a glanceable
        briefing with detail behind disclosure. Optional local or BYOK AI receives
        a guarded read-only bundle and can disappear without changing the match
        analysis by a single digit. Selected-source research captures exact text and hashes,
        then routes untrusted candidates into local correction review.
      </div>
      <div class="gv-mini-steps" aria-label="AI read choices">
        <span><b>Fast</b> small local model, short grounded read</span>
        <span><b>Deep</b> larger model, connected scenarios, usually 5–8 minutes</span>
        <span><b>Ollama setup</b> install models in-app with progress and cancel</span>
      </div>
      <ul class="gv-check-list">
        <li>Every displayed number is engine-owned</li>
        <li>Model voices stay separate instead of being averaged</li>
        <li>Research fetches only a source you selected and never makes it authoritative</li>
        <li>Unsupported AI output fails back to deterministic analysis only</li>
      </ul>

    <div class="gv-chapter-copy gv-chapter-copy--wide">
      <p class="gv-kicker">Private by architecture, not by promise</p>
      <h2 id="gv-architecture-title" class="gv-chapter-title">A local app with<br/><em>one numeric authority.</em></h2>
      <div class="gv-body">
        Tauri supervises a token-protected loopback service. React presents
        validated views. FastAPI serves typed contracts. The deterministic Python
        core is the only place a forecast can be made; optional narration stays
        outside that write path.
