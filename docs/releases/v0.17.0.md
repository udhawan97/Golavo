# Golavo v0.17.0 — Proof you can carry, context that knows its limits

Golavo 0.17.0 strengthens the trust chain around a forecast and keeps domestic league
state moving without pretending one source is several. Official builds now authenticate
the exact pack manifests they load, every forecast can travel with an offline-verifiable
proof bundle, and the release checksum ledger has its own detached signature.

The Match Cockpit also becomes more explicit about what it knows: readiness names the
actual evidence boundary, verified-generation diffs explain deterministic probability
movement, and the Reference Desk shows only revision-pinned venue facts. It deliberately
withholds current-manager claims until Golavo has a revision-pinned tenure source.

## What is new

- **Signed official pack manifests.** Stable packaging signs every active pack manifest
  with Golavo's existing Ed25519 updater identity. The frozen sidecar verifies the key id,
  payload signature, and trusted-comment signature before reading a bundled pack. Missing
  or altered signatures fail closed. Source-mode and runtime-generated refresh packs keep
  their separate reproducible manifest/receipt boundary.
- **Portable forecast proof.** Forecast Detail downloads the sealed/scored/voided lineage,
  source descriptors, matching exact manifest bytes when available, contract versions, and
  a canonical proof hash. `golavo verify-proof proof.json` validates the bundle offline;
  unavailable manifests stay labelled `descriptor-only` rather than being implied.
- **Eight-source continuity.** Refresh now covers martj42 internationals, World Cup and
  football JSON, and the exact current-season file in each of five OpenFootball country
  repositories. Domestic generation is atomic, preserves verified history, and keeps the
  previous generation on absence, malformed data, schedule gaps, or result conflicts.
- **Reference Desk.** Match pages show only evidence-bundle venue identity/capacity with
  the exact source revision. Manager tenure is intentionally not shown without a pinned
  source that can prove the claim at the match cutoff.
- **Forecast readiness.** A compact checklist exposes generation fingerprint, kickoff
  precision, historical support, the seal path, and missing lineups/injuries/xG. It is an
  evidence-readiness display, not a confidence or accuracy score.
- **Verified-generation diff.** After the verified index fingerprint changes, the cockpit
  shows model-voice 1X2 movement from the last locally viewed generation. The comparison
  is deterministic local display state and cannot call AI or write a forecast.
- **Conditional season scenarios.** Pick one remaining domestic fixture and assign a
  hypothetical score to rerun Ratings, Goals, and baseline season voices. The result is
  stamped hypothetical-only, never persisted, never sealed, and never model input. Reset
  returns to the canonical verified outlook.

## Trust boundaries

- Installers remain OS-unsigned and not notarized; Gatekeeper or SmartScreen may warn on
  first install. Updater payloads, active bundled pack manifests, and the aggregate checksum
  ledger are separately authenticated with Golavo's pinned release identity.
- One domestic Football.TXT result is still one source. Club settlement continues to wait
  for an independent source to agree.
- Runtime refresh is while-open and consent-gated. There is no daemon or closed-app claim.
- Reference facts are not model inputs. Current managers, lineups, injuries, suspensions,
  and observed xG remain unavailable.
- A conditional season scenario is not a forecast artifact and cannot enter calibration.

## Verification

The release gate rebuilds the deterministic match index, validates every contract, source,
manifest, artifact, correction, research, and license boundary, and runs the full Python,
TypeScript, browser, docs, Rust, and packaged-sidecar checks. The published macOS and Windows
assets are then checked against the signed checksum ledger; the installed macOS app is opened
and exercised as the final shipped surface.
